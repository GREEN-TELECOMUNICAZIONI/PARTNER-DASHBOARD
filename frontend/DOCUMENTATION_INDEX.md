## Contributing to Documentation

### When to Update Documentation

- Adding new features → Update [FEATURES.md](./FEATURES.md)
- Changing architecture → Update [ARCHITECTURE.md](./ARCHITECTURE.md)
- Adding API endpoints → Update [API_INTEGRATION.md](./API_INTEGRATION.md)
- New deployment process → Update [DEPLOYMENT_CHECKLIST.md](./DEPLOYMENT_CHECKLIST.md)
- New scripts/commands → Update [QUICK_REFERENCE.md](./QUICK_REFERENCE.md)
- Any significant change → Update [README.md](./README.md)

### Documentation Standards

1. **Keep it updated:** Documentation should always reflect current state
2. **Be clear:** Write for someone who doesn't know the project
3. **Use examples:** Show, don't just tell
4. **Keep it organized:** Use consistent structure
5. **Link between docs:** Reference related documentation
